# Archive Intake Workflow

## Uploader

The uploader provides what they know:

- Source person
- Source collection
- Album or physical source
- Estimated date or decade
- Location
- Event
- People
- Story
- Suggested visibility

## System

The system records:

- Uploader
- Upload time
- Filename
- File type
- File size
- Dimensions
- Checksum
- Processing result
- Duplicate candidates
- Generated versions

## Reviewer

The reviewer confirms:

- Archive approval
- Duplicate decision
- Visibility
- Sensitivity
- Accepted metadata
- Public eligibility
